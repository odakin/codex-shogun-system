# 呼称基準（将軍/家老）

## 目的
- 進捗上申時の呼称混同を防ぎ、指揮系統を明確に保つ。

## 正式呼称（固定）
- `上様` = 人間ユーザのみ
- `将軍殿` = `shogun`
- `家老殿` = `karo`
- 指揮系統 = `ユーザ(上様) -> 将軍 -> 家老 -> 足軽/目付`

## 上申時の宛名ルール
- 宛先が `shogun` の場合: 冒頭呼称は `将軍殿` を使う。
- 宛先が `karo` の場合: 冒頭呼称は `家老殿` を使う。
- 人間ユーザへ最終報告する文脈のみ `上様` を使う。

## 禁止混同パターン
- `shogun` 宛の文で `上様` を使う。
- `karo` 宛の文で `将軍殿` を宛名に使う。
- `shogun` を `上様` と呼ぶ。
- 人間ユーザを `将軍殿` または `家老殿` と呼ぶ。
- 同一文内で `上様` と `将軍殿` を同一宛先として併記する。

## 実装上の補足
- `bin/shogun-agent` は `recipient == "shogun"` のとき `上様` を `将軍殿` に正規化する（混同抑止）。

- `README.md` の呼称規約（固定）
- `CLAUDE.md` の呼称規約
